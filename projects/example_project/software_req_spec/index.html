    Software Requirements Specification

    REST API for Service Router

    1.Введение

    1.1 Цель документа

    Данный документ описывает функциональные и нефункциональные требования к REST API сервиса маршрутизации SWIFT FIN сообщения

    Сервис обеспечивает: - прием SWIFT сообщений через REST API - публикацию сообщений во входящий Kafka topic - асинхронную маршрутизацию сообщений - передачу обработанных сообщений в выходящий Kafka topic - получение статуса маршрутизации "routed"

    1.2 Область применения

    Сервис используется для маршрутизации финансовых сообщений SWIFT между внутренними банковскими системами.

    2.Общее описание

    2.1 Архитектурный контекст

    Client System -> POST /swift/messages -> Routing REST Service -> Kafka Topic (swift.inbound) -> Routing Worker -> Kafka Topic (swift.outbound) -> Target System

    2.2 Основные компоненты

    Компонент Назначение
    REST Routing API Приём SWIFT сообщений
    Inbound Topic Очередь входящих сообщений
    Routing Worker Определение маршрута
    Outbound Topic Очередь исходящих сообщений
    Status Store Хранение статусов обработки

    3.Фунциональные требования

    3.1 Интерфейс системы

    3.1.1 Метод: POST /swift/v1/messages

    3.1.1.1 Описание

    Метод принимает SWIFT сообщение для последующей асинхронной маршрутизации

    3.1.1.2 HTTP Запрос
    Параметр Значение
    Метод POST
    URL /swift/v1/messages
    Content-Type application/json
    3.1.1.3 Тело запроса

    { "messageId": "string", "swiftType": "MT103", "payload": "string" }

    3.1.1.4 Обработка запроса
    1. Сервис валидирует SWIFT сообщение
    2. Генерирует routingId по алгоритму snowflake
    3. Формирует routing event
    4. Публикует сообщение в Kafka topic: swift.inbound
    5. Сохраняет статус: ROUTED
    3.1.1.5 Ответ
    Код Описание
    202 Accepted Сообщение принято

    { "routingId": "uuid", "statusUrl": "/swift/messages/{routingId}" }

    3.1.2 Метод: GET /swift/messages/{routingId}

    3.1.2.1 Описание

    Метод возвращает текущий статус маршрутизации SWIFT сообщения

    3.1.2.2 Ответ

    В обработке

    | Код | 202 Accepted |

    { "status": "IN_PROGRESS" }

    Завершено

    | Код | 200 OK |

    { "status": "ROUTED", "outboundTopic": "swift.outbound" }

    Ошибка

    { "status": "FAILED", "errorCode": "ROUTING_ERROR" }

    4. Kafka Integration Requirements

    4.1 Inbound Topic

    Параметр Значение
    Topic Name swift.inbound
    Key routingId
    Value SWIFT Routing Event

    4.2 Outbound Topic

    Параметр Значение
    Topic Name swift.outbound
    Key routingId
    Value Routed SWIFT Message

    5. Статусы обработки

    Статус Описание
    ROUTED Сообщение маршрутизировано
    VALIDATED Прошло валидацию
    IN_PROGRESS В маршрутизации
    ROUTED Отправлено в outbound
    FAILED Ошибка маршрутизации

    6. Нефункциональные требования

    Требование Значение
    Latency POST ≤ 150 мс
    Throughput ≥ 1000 msg/sec
    Delivery At-least-once
    Availability 99.99%
    Security TLS + OAuth2 + JWT

    7. Ограничения

    • Kafka cluster должен быть доступен
    • Routing Worker должен быть подписан на topic: swift.inbound
    • ACL kafka User:consumer-user, read swift.inbound
